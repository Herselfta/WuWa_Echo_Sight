# Roadmap

WuWa Echo Sight 的未来发展方向与优先级规划。

> 标记说明：🟢 近期可行 · 🟡 中期目标 · 🔵 远期愿景

---

## Phase 1 — 核心体验完善 🟢

当前版本已实现基础功能闭环，Phase 1 专注于打磨日常使用的细节和可靠性。

### 数据安全与恢复
- [ ] **自动备份**：应用启动/关闭时自动保存数据库快照（保留最近 N 份）
- [ ] **手动导出/导入**：一键导出全量 SQLite 数据库文件，支持恢复
- [ ] **撤销录入**：支持最近一次强化事件的一键撤销

### UI / UX 优化
- [ ] **深色/浅色主题**：跟随系统偏好或手动切换
- [ ] **响应式布局**：适配更大/更小的窗口尺寸
- [ ] **键盘快捷键**：录入场景支持 Tab / Enter 快速操作
- [ ] **声骸搜索/筛选**：在声骸池按昵称、状态、期望词条模糊搜索
- [ ] **录入确认反馈**：录入成功后短暂高亮变更的数据项

### 概率分析增强
- [ ] **每日趋势图**：将全局概率按日期展开，观察分布随时间的变化
- [ ] **多声骸对比**：选中多颗声骸，并排对比其概率曲线
- [ ] **理论 vs 实际**：将实测概率与理论等概率线叠加对比

---

## Phase 2 — 社区与效率 🟡

在核心体验稳固后，提升经验分享和操作效率。

### 数据共享
- [ ] **匿名统计上报**（opt-in）：贡献脱敏后的概率数据到公共池，供社区参考
- [ ] **社区概率仪表盘**：聚合后的概率统计数据可视化
- [ ] **配置导出分享**：导出期望预设为 JSON/链接，方便配装讨论时分享

### 智能辅助
- [ ] **强化建议**：根据当前声骸状态和期望配置，推荐「继续还是弃置」
- [ ] **期望模板库**：内置主流角色/套装的推荐期望配置
- [ ] **批量录入**：支持一次提交多条强化数据（减少切换操作）

### 工程质量
- [ ] **前端单元测试**：Vitest + React Testing Library
- [ ] **后端集成测试**：Rust 端命令层测试
- [ ] **CI/CD**：GitHub Actions 自动构建 + 类型检查 + 版本发布
- [ ] **自动更新**：Tauri updater 插件，应用内检测新版本

---

## Phase 3 — 平台拓展 🔵

面向更广泛的用户群体和使用场景。

### 跨端支持
- [ ] **macOS 构建**：CI 产出 `.dmg` 安装包
- [ ] **Linux 构建**：CI 产出 `.AppImage` / `.deb`
- [ ] **移动端适配**（探索性）：Tauri v2 移动端能力，iOS/Android 基础支持

### OCR 自动录入
- [ ] **截图识别**：识别游戏内强化界面截图，自动填充词条和档位
- [ ] **OBS 集成**（实验性）：实时捕获游戏画面并自动录入

### 数据生态
- [ ] **Webhook 通知**：强化达标/概率异常时推送到 Discord/微信
- [ ] **API 接口**：暴露本地 HTTP API 供其他工具集成
- [ ] **仪表盘嵌入**：生成可分享的只读数据仪表盘 URL

### 国际化
- [ ] **i18n 框架**：提取所有硬编码文案到语言文件
- [ ] **英语支持**：面向国际版玩家的英文界面
- [ ] **日语支持**：面向日服玩家

---

## 技术债务 & 持续改进

这些不属于某个 Phase，但应贯穿整个开发过程：

- [ ] **提取公共组件**：将 `STAT_ABBR_MAP`、`formatScaledValue()` 等从各页面提取为共享模块
- [ ] **拆分大组件**：RecordPage (~1100+ 行) 和 EchoPoolPage (~2600+ 行) 需拆分子组件
- [ ] **样式模块化**：将 `App.css` (1700+ 行) 按页面拆分为 CSS Modules 或 scoped styles
- [ ] **类型安全加强**：消除 `as` 类型断言，用 Zod schema 验证 IPC 返回值
- [ ] **性能优化**：大量声骸时的虚拟列表、memo 优化、减少不必要的 re-render
